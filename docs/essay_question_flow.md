# 简答题从出题到评分完整流程

## 流程图

```mermaid
flowchart TD
    A[教师出题] --> B{选择题型为简答题?}
    B -->|是| C[填写题目内容、分值、参考答案、关键词列表]
    B -->|否| D[按客观题流程出题]
    C --> E[保存题目到题库]
    E --> F[考试发布，学生参加考试]
    F --> G[学生答题: 输入文本答案]
    G --> H[学生提交试卷]
    H --> I[系统自动批改]
    I --> J{是否有关键词设置?}
    J -->|是| K[计算关键词命中数量, 按比例给出初始分数]
    J -->|否| L[初始分数设为0, 待人工批改]
    K --> M[记录初始得分]
    L --> M
    M --> N[考试记录状态变为"待批改"]
    N --> O[教师查看学生答卷]
    O --> P{需要调整分数?}
    P -->|是| Q[手动调整分数]
    P -->|否| R[确认分数]
    Q --> R
    R --> S{所有简答题都批改完成?}
    S -->|是| T[考试记录状态变为"已完成", 总分更新]
    S -->|否| U[继续批改其他简答题]
    U --> O
    T --> V[学生查看最终成绩]
```

## 流程说明

### 1. 出题阶段
- 教师在出题页面选择"简答题"题型
- 填写题目内容、分值
- 填写**参考答案**（供教师批改时参考）
- 填写**关键词列表**（多个关键词用逗号分隔，系统自动评分使用）
- 保存题目，简答题会自动保存参考答案和关键词到数据库

### 2. 答题阶段
- 学生进入考试，简答题显示为多行文本输入框
- 学生在文本框中输入答案内容
- 系统自动保存学生的文本答案，提交时完整上传

### 3. 自动评分阶段
- 学生提交试卷后，系统自动处理：
  - 对于客观题（单选、多选、判断），按原有逻辑直接判断对错并给分
  - 对于简答题：
    - 先检查题目是否有关键词设置
    - 如果有关键词，将学生答案转为小写，逐一匹配关键词
    - 计算公式：`得分 = 题目总分 * (命中关键词数量 / 总关键词数量)`，结果四舍五入取整
    - 如果没有关键词，初始得分设为0，等待教师手动批改
- 如果考试包含简答题，考试记录状态会标记为**待批改**；如果全是客观题，直接标记为**已完成**

### 4. 人工调整阶段
- 教师在考试记录页面查看学生答卷
- 对于简答题，可以看到学生的答案、参考答案、关键词以及系统给出的初始分数
- 教师可以根据答案质量手动调整分数
- 每修改一个简答题的分数，系统会自动重新计算考试总分
- 当所有简答题都批改完成后，考试记录状态自动变为**已完成**

### 5. 成绩查看
- 学生可以在成绩记录中看到简答题的得分和教师最终给出的总分
- 如果考试还在批改中，学生看到的状态为"批改中"

## 技术实现要点
- 数据库新增字段：`question` 表增加 `reference_answer`（TEXT）和 `keywords`（VARCHAR(500)）字段
- 题型枚举：新增类型4表示简答题
- 自动评分逻辑：基于关键词匹配度计算初始分数
- 考试状态：新增"待批改"状态（status=1），表示有简答题需要人工审核
- 新增接口：`PUT /api/record/answer/{answerId}/score` 用于教师手动调整分数
